# Инструкция по деплою платформы репетитора

Всего 5 шагов. Займёт ~20 минут.

---

## Шаг 1. Создать базу данных в Supabase

1. Зайди на [supabase.com](https://supabase.com) → **Start your project** → войди через GitHub
2. Нажми **New Project**, придумай имя (например `tutor`), придумай пароль базы, выбери регион **Frankfurt** (ближе всего)
3. Подожди 1-2 минуты пока проект создаётся
4. В левом меню нажми **SQL Editor** → **New query**
5. Скопируй всё содержимое файла `supabase-schema.sql` и вставь в редактор → нажми **Run**
6. Убедись что написано `Success. No rows returned`

**Сохрани ключи** — они понадобятся на шаге 4:
- Иди в **Settings → API**
- Скопируй `Project URL` → это `NEXT_PUBLIC_SUPABASE_URL`
- Скопируй `anon public` → это `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- Скопируй `service_role secret` → это `SUPABASE_SERVICE_ROLE_KEY`

---

## Шаг 2. Загрузить код на GitHub

1. Зайди на [github.com](https://github.com) → войди или зарегистрируйся
2. Нажми **+** → **New repository**, назови `tutor-platform`, нажми **Create repository**
3. Скачай [GitHub Desktop](https://desktop.github.com/) если ещё нет
4. В GitHub Desktop: **File → Add local repository** → выбери папку `tutor-platform`
5. Нажми **Commit to main** → **Push origin**

Код теперь на GitHub.

---

## Шаг 3. Создать Telegram-бота

1. Открой Telegram → найди **@BotFather**
2. Напиши `/newbot`
3. Придумай имя (например `Мой репетитор`) и username (например `my_tutor_bot`)
4. BotFather пришлёт токен вида `123456789:AAF...` → сохрани его

**Узнай свой Telegram ID:**
1. Напиши боту **@userinfobot** — он пришлёт твой ID (число)
2. Сохрани его

---

## Шаг 4. Задеплоить на Vercel

1. Зайди на [vercel.com](https://vercel.com) → **Continue with GitHub**
2. Нажми **Add New → Project** → выбери репозиторий `tutor-platform` → **Import**
3. В разделе **Environment Variables** добавь все переменные:

| Имя | Значение |
|-----|----------|
| `NEXT_PUBLIC_SUPABASE_URL` | URL из Supabase Settings → API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon key из Supabase |
| `SUPABASE_SERVICE_ROLE_KEY` | service_role key из Supabase |
| `TELEGRAM_BOT_TOKEN` | токен от @BotFather |
| `TELEGRAM_ALLOWED_USER_ID` | твой Telegram ID (число) |

4. Нажми **Deploy** и подожди ~2 минуты
5. Vercel покажет твой URL — например `https://tutor-platform-abc.vercel.app`

---

## Шаг 5. Подключить Telegram-бота

Telegram-бот работает через webhook — он отправляет сообщения в твоё приложение.

Открой браузер и вставь эту ссылку (замени значения):

```
https://api.telegram.org/botТВОЙ_ТОКЕН/setWebhook?url=https://ТВОЙ_URL.vercel.app/api/telegram
```

Пример:
```
https://api.telegram.org/bot123456789:AAF.../setWebhook?url=https://tutor-platform-abc.vercel.app/api/telegram
```

Если всё успешно, браузер покажет:
```json
{"ok":true,"result":true,"description":"Webhook was set"}
```

**Готово!** Напиши боту `/help` — он должен ответить списком команд.

## Как пользоваться ботом

```
/today                          — занятия сегодня
/week                           — занятия на этой неделе
/debts                          — кто должен

/pay Анна 2000                  — записать оплату
/pay Анна 2000 за январь        — оплата с заметкой

/student Анна                   — баланс и ближайшее занятие

/cancel Анна 30.06              — отменить занятие 30 июня
/reschedule Анна 30.06 на 02.07 14:00  — перенести занятие
```

## Обновление приложения

Когда я пришлю обновлённый код — просто перетащи новые файлы в папку проекта и в GitHub Desktop нажми **Commit → Push**. Vercel сам пересоберёт и обновит сайт за 2 минуты.

---

## Если что-то не работает

- **Сайт не открывается** — подожди 2-3 минуты после деплоя
- **Бот не отвечает** — проверь, правильно ли вставлен токен и URL в webhook
- **Данные не сохраняются** — проверь переменные окружения в Vercel Settings → Environment Variables
- **Ошибка в консоли браузера** — напиши мне, разберёмся
